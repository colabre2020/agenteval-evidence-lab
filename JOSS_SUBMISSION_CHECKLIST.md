# JOSS Submission Checklist

## Must-have before submission
- [ ] Repository is public and clonable without login
- [ ] OSI-approved license is present (MIT is OK)
- [ ] README contains install, run, and validation instructions
- [ ] Paper files (`paper.md` and `paper.bib`) are in the repository root at submission time
- [ ] Software has clear research use and is not a one-off script
- [ ] Issue tracker is enabled and public

## JOSS paper quality checks
- [ ] Summary is understandable to non-specialists
- [ ] Statement of need clearly explains problem + users
- [ ] State of the field compares with related tools
- [ ] Software design explains architectural trade-offs
- [ ] Research impact statement includes concrete evidence
- [ ] AI usage disclosure is explicit and accurate

## Final submission steps
1. Go to http://joss.theoj.org/papers/new
2. Submit repository URL and article metadata
3. Participate in public GitHub review thread
4. After acceptance: create tagged release + archive (Zenodo/Figshare) + DOI
